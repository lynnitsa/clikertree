Например, int numberHome = 10; Если дробное, то ставится double. Тип boolean называется логическим и имеет всего два значения — true (англ. «истина») и false (англ. «ложь»). Их можно сравнить со словами «да» и «нет». Если на какой-то вопрос можно ответить «да», то это true, если «нет» — false. Например, значение типа boolean появляется при сравнении чисел: 3 < 5 // Три меньше пяти? Да, это так -> true 4 > 8 // Четыре больше восьми? Нет, это ложь -> false --------------- boolean isMore = 3 > 4; System.out.println("Значение переменной isMore = " + isMore); В результате будет напечатано: Значение переменной isMore = false Кстати, мы не случайно назвали переменную isMore. Дело в том, что название булевых или логических переменных — типа boolean — принято начинать с префикса is (англ. «является»). Например, переменную с именем isEven принято использовать для проверки чётности, а isNegative — того, что число отрицательное. Это не жёсткое правило, но по традиции переменные называют именно так, чтобы проще было читать и понимать код. Так что если встретили переменную с именем, начинающуюся с is, то, скорее всего, она будет иметь тип boolean. Проверить, равны ли числа друг другу, можно с помощью оператора ==. Это напечатанные подряд два символа «равно». Оператор возвращает логический результат — значение false или true: 3 == 4 // Три равно четырём? Нет -> false Такое выражение тоже можно сохранить в переменной: boolean isEqual = 3 == 4; Обратите внимание, что одиночный знак = выполняет присваивание, а двойной == — проверку на равенство. Можно проверить числа и на неравенство. Для этого в Java есть оператор !=, который возвращает true, если значения не равны, и false в противоположном случае: boolean isNotEqual = 3 != 4; // Три не равно четырём? Да -> true Операторы сравнения «больше или равно» >= и «меньше или равно» <= также возвращают значение типа boolean: int three = 3; int six = 6; boolean isMoreOrEqual = three >= 5; // Три больше или равно пяти? Нет -> false boolean isLessOrEqual = six <= 8; // Шесть меньше или равно восьми? Да -> true При сравнении строк оператор == работает не корректно, все дело в том, что числа и строки в Java представлены по-разному. Для сравнения строк есть особый метод — equals (англ. «сравнивать»). Его синтаксис выглядит так: boolean isEqual = "Привет".equals("Здравствуйте"); // Строки равны? Нет -> false Метод equals сравнивает строку до точки с той, что написана в круглых скобках. Если они равны, то будет возвращено true, иначе — false. Строки можно указывать в кавычках, или сравнивать между собой переменные с типом String: String str1 = "Какой-то текст"; String str2 = "Какой-то " + "текст"; boolean isSameString = str1.equals(str2); // Переменная isSameString = true isSameString = str1.equals("Другая строка"); // Теперь переменная isSameString = false ------------------------ ОШИБКИ: Uncaught SyntaxError: Unexpected identifier -Это значит, что в коде появилась неизвестная переменная, команда или объект, о которых браузер не знает.
